Mossboard is about project visibility, but without turning planning into theatre.

The point is to get a cleaner view across active work, decisions, and bottlenecks, especially when there are multiple repos moving at once.

## Current focus

- define the MVP for cross-project visibility
- make issue browsing and task context easier to navigate
- keep the system lightweight enough to actually use daily
- support real execution rather than decorative PM workflows

## Why it matters

Once the number of active repos climbs, the cost of poor visibility climbs with it. Mossboard is an attempt to fix that at the operating-system layer for project work.
