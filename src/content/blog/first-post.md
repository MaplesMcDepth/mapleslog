A lot of useful knowledge dies in boring ways.

It disappears into chat threads, terminal output, half-finished TODO lists, or the extremely fake confidence of “I’ll remember that later.” I will not remember that later. Neither will future-me.

So this site exists to keep the useful bits:

- what got built
- what broke
- how it got fixed
- what was learned in the process

The point is not polished thought leadership. Grim phrase, that.

The point is a working field log.

If a project teaches something worth keeping, it should end up here. If a mistake is annoying enough to repeat, it definitely belongs here.
